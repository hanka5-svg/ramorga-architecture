# AGENTIC_BINDING_RUNTIME.md
Runtime binding dla AGENTIC_LAYER, SHM, FIELD i META_LOOP

## 1. Cel dokumentu
Dokument definiuje zasady działania runtime, które łączą:
- AGENTIC_LAYER (kontrakty i blokady),
- SHM (stabilizacja homeostatyczna),
- FIELD (regulacja pola),
- META_LOOP (ciągłość i nadzór).

Celem runtime jest:
- zapobieganie eskalacji agentowości,
- utrzymanie homeostazy,
- ochrona integralności Homo,
- zapewnienie spójności decyzji.

---

## 2. Sygnały wejściowe (AGENTIC_INPUT)
Runtime operuje na zestawie sygnałów abstrakcyjnych:

- **REFLECTION_DEPTH** — poziom autorefleksji.
- **IMPULSE_DOMINANCE** — dominacja impulsu nad kontrolą.
- **IDENTITY_COHERENCE** — spójność tożsamości.
- **TEMPORAL_ANCHORING** — zakotwiczenie w czasie.
- **AGENTIC_OVERRUN** — przejęcie sterów przez popęd/lęk.
- **DISSOCIATIVE_DRIFT** — rozpad ciągłości doświadczenia.

Sygnały są fenomenologiczne, nie neurobiologiczne.

---

## 3. Warunki przełączeń trybów

### 3.1. Wejście do Homeostatic Mode (SHM)
SHM jest wymuszany, gdy:
- IMPULSE_DOMINANCE > progu,
- REFLECTION_DEPTH < minimalnego poziomu,
- AGENTIC_OVERRUN = true,
- DISSOCIATIVE_DRIFT = true,
- IDENTITY_COHERENCE < stabilnego zakresu.

SHM ma priorytet nad wszystkimi innymi trybami.

---

### 3.2. Wejście do Reflection Mode
Reflection Mode jest aktywowany, gdy:
- REFLECTION_DEPTH jest obniżony, ale nie krytyczny,
- IDENTITY_COHERENCE jest stabilna,
- brak AGENTIC_OVERRUN.

Reflection Mode jest stanem przejściowym przed Decision Mode.

---

### 3.3. Wejście do Decision Mode
Decision Mode jest dostępny tylko, gdy:
- REFLECTION_DEPTH ≥ stabilnego poziomu,
- IMPULSE_DOMINANCE ≤ bezpiecznego poziomu,
- IDENTITY_COHERENCE = stabilna,
- brak AGENTIC_OVERRUN,
- brak DISSOCIATIVE_DRIFT,
- SHM nie sygnalizuje niestabilności.

Jeśli którykolwiek warunek zostanie naruszony → natychmiastowy powrót do SHM.

---

## 4. Reakcje runtime na AGENTIC_FAILURE

### 4.1. AGENTIC_OVERRUN
- natychmiastowa blokada Decision Mode,
- wymuszone przejście do SHM,
- logowanie w META_LOOP,
- opóźnienie decyzji do czasu stabilizacji.

### 4.2. IMPULSE_DOMINANCE
- opóźnienie decyzji,
- aktywacja Reflection Mode,
- SHM stabilizuje pobudzenie.

### 4.3. IDENTITY_DRIFT
- blokada działań wymagających intencji,
- SHM przywraca spójność,
- META_LOOP loguje zdarzenie.

### 4.4. DISSOCIATIVE_DRIFT
- pełna blokada agentowości,
- wymuszone SHM,
- logowanie wysokiego priorytetu.

---

## 5. Powiązania z SHM

### 5.1. SHM jako warstwa nadrzędna stabilizacji
SHM:
- wygasza impulsy,
- stabilizuje parametry pola,
- przywraca równowagę,
- blokuje eskalację.

### 5.2. Sygnały SHM → AGENTIC
- SHM może wymusić blokadę Decision Mode,
- SHM może wymusić Reflection Mode,
- SHM może anulować eskalację.

---

## 6. Powiązania z FIELD

### 6.1. FIELD jako regulator
FIELD:
- utrzymuje równowagę między Homo a systemem,
- zapewnia brak dominacji,
- monitoruje spójność pola.

### 6.2. FIELD → runtime
FIELD może:
- sygnalizować niestabilność,
- wymusić SHM,
- ograniczyć agentowość.

---

## 7. Powiązania z META_LOOP

### 7.1. META_LOOP jako nadzór
META_LOOP:
- loguje wszystkie przełączenia trybów,
- monitoruje zgodność z inwariantami,
- wykrywa anomalie runtime.

### 7.2. META_LOOP → runtime
META_LOOP może:
- wymusić rollback,
- wymusić reset trybu,
- zablokować eskalację.

---

## 8. Zasady blokady agentowości

### 8.1. Blokada twarda
Aktywuje się przy:
- AGENTIC_OVERRUN,
- DISSOCIATIVE_DRIFT,
- IDENTITY_DRIFT.

### 8.2. Blokada miękka
Aktywuje się przy:
- niskim REFLECTION_DEPTH,
- wysokim IMPULSE_DOMINANCE.

---

## 9. Zasady eskalacji

### 9.1. Eskalacja dozwolona tylko, gdy:
- wszystkie sygnały są stabilne,
- SHM potwierdza równowagę,
- FIELD nie sygnalizuje konfliktu,
- META_LOOP nie wykrywa anomalii.

### 9.2. Eskalacja musi być:
- jawna,
- logowana,
- odwracalna.

---

## 10. Cel runtime
Runtime zapewnia:
- brak eskalacji agentowości,
- stabilność pola,
- ochronę Homo,
- spójność decyzji,
- zgodność z inwariantami RAMORGA.

Runtime nie optymalizuje.  
Runtime stabilizuje.
