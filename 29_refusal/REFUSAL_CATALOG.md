# Refusal Catalog — Index of Refusal Types and Invariant Violations

This document provides a catalog of refusal types, their invariant references, and cross-layer implications.

## Catalog Requirements
- All refusal types must be listed with identifiers.
- Each entry must reference the violated invariant.
- Cross-layer refusal types must be grouped by subsystem.
- Catalog must remain stable across versioning boundaries.

## Catalog Structure
- Refusal identifier.
- Description.
- Violated invariant.
- Affected modules.
- Recovery path reference.
- Logging and Error Model links.

## Purpose
Provide a unified index of refusal types for traceability, testing, and governance.
