# ADR Template — Deterministic Structure for Architecture Decisions

This template defines the required structure for all ADRs in RAMORGA.

## Required Sections
- **Context** — Deterministic description of the architectural situation.
- **Decision** — Explicit, invariant-aligned choice.
- **Alternatives** — Deterministic comparison of rejected options.
- **Invariant Impact** — Which invariants are affected and how.
- **Consequences** — Expected system-wide effects.
- **Version Links** — Related version transitions.
- **Tests** — Architecture Tests validating the decision.

## Purpose
Ensure all ADRs follow a unified, deterministic structure.
