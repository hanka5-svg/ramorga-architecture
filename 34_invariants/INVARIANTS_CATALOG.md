# Invariants Catalog — Index of Global and Local Architectural Invariants

This document provides a complete catalog of invariants governing RAMORGA.  
Invariants define the non-negotiable rules that all modules, layers, and transitions must obey.

## Catalog Requirements
- All invariants must be listed with stable identifiers.
- Each invariant must include a precise, deterministic definition.
- Cross-layer invariants must be grouped by subsystem.
- Violations must map to refusal, recovery, or error-model categories.
- Catalog must remain stable across version transitions.

## Catalog Structure
- Invariant identifier.
- Deterministic definition.
- Affected modules and layers.
- Violation consequences.
- Links to diagrams and tests.

## Purpose
Provide a unified, authoritative index of invariants that define RAMORGA’s architectural identity.
